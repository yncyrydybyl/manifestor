# Contributing

Thanks for contributing.

## Development setup

```bash
git clone https://github.com/yncyrydybyl/manifestor.git
cd manifestor
make test
make build
```

## Before opening a PR

- Keep changes focused
- Add or update tests
- Update docs if behavior changed
- Make sure CI passes: `make test && make lint`

## Commit style

Use short, clear commit messages.

## Pull requests

Please include:
- What changed
- Why it changed
- How it was tested
